# with-ai-development-foundation

AIを単なる補助ツールではなく、共同開発者として活用するための開発手法・運用ルール・実践知を整理する公開資料リポジトリです。

> 本リポジトリの資料は、主に日本語で作成しています。

## このリポジトリについて

本リポジトリは、「AIネイティブな開発基盤」に関する研究・実践の途中成果を、社内共有・公開共有しやすい形で整理したものです。

ここでいう「開発基盤」は、特定のツールやアプリケーションだけを指すものではありません。AIと人間が共同で調査、設計、実装、レビュー、運用を進めるための考え方、ワークフロー、ルール、テンプレート、失敗回避策などを含みます。

## 現在の位置づけ

このリポジトリの内容は、研究・整理の途中段階です。

記載内容は完成済みのベストプラクティスではなく、実践、調査、検証を通して継続的に見直されます。

研究の進捗により、以下のような変更が行われる可能性があります。

- 既存ドキュメントの内容更新
- 新しい観点・章・資料の追加
- ディレクトリ構成や文書構成の整理
- 一部の表現、前提、結論の見直し

## ターゲット層

主な対象読者は以下のような人です。

- AIを開発プロセスに取り入れたいエンジニア
- AIエージェントやLLMを使った開発運用を整理したいチームリーダー、開発マネージャー
- 個人または小規模チームで、AI活用のルール・テンプレート・ナレッジベースを作りたい人
- 「AIを使う」段階から一歩進んで、「AIと一緒に開発する」体制に関心がある人

## 現時点で扱う主なテーマ

現時点では、以下のようなテーマを中心に整理していく予定です。

- AIと人間の役割分担
- AIに渡すコンテキストの設計
- AIを前提とした開発フロー、レビュー、タスク分割
- AI活用時のセキュリティ、情報管理、失敗回避
- チームや個人で再利用できるルール、テンプレート、運用パターン

今後の研究の進捗に応じて、扱うテーマは追加・変更される可能性があります。

## このリポジトリが目指すもの

このリポジトリは、AI活用に関する抽象的な一般論だけをまとめるものではありません。

実際の開発・調査・設計・運用の中で得られた知見を、他の人が読んでも再利用しやすい形に整理することを目指しています。

特に、以下のような問いを扱います。

- AIに何を任せ、何を人間が判断するべきか
- AIが作業しやすい開発環境や情報構造とは何か
- AIを使うことで、従来の開発プロセスはどう変わるのか
- AI活用における失敗やリスクを、どのように事前に減らせるか
- 個人の試行錯誤を、チームで使える知見に変換するにはどうすればよいか

## 参考資料・出典について

この研究では、公開情報、公式ドキュメント、コミュニティ上の事例、個人の実践記録などを参考にしています。

そのため、当面は本文中の内容を「確定的な外部標準」ではなく、「現在の研究・実践に基づく整理」として扱ってください。

参考資料や出典の整理が進み次第、関連ドキュメントまたは各章に追記していきます。
